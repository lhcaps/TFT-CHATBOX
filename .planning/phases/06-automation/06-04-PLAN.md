      - **False branch:** No-op / log "No new patch detected"
    - Node 4 (Error Handler / On Error): HTTP POST Discord webhook with red embed notification
    - Discord embed: color 15158332, title "🚨 TFT Patch Ingest Failed", fields: patch version, error message, action, timestamp
    - `active: false` (user activates manually)

Workflow flow:
1. Cron every 6 hours (0 */6 * * *)
2. Fetch Riot latest version and backend cached version in parallel
3. IF node compares — if versions differ:
   - POST to /api/ingest/tft-static (with Bearer auth via `tftBackendApi`)
   - On error → Discord notification (red embed)
4. IF versions match: log "No new patch detected"

The Discord embed format per D-14: color 15158332 (red), title with emoji, fields: patch, error, action, timestamp.

Note: The `Trigger TFT Ingest` node references `tftBackendApi` credential (same as obsidian workflow). The Discord node references `discordWebhook` credential which stores the full Discord webhook URL — no env var needed.

This task requires manual action in the n8n web UI to create the Discord webhook credential:

**Step 1: Get Discord webhook URL**
1. Open Discord → right-click your channel → Edit Channel
2. Go to Integrations → Webhooks → New Webhook
3. Name it "TFT Copilot Alerts" → Copy Webhook URL
4. URL format: `https://discord.com/api/webhooks/{webhook_id}/{webhook_token}`

**Step 2: Create discordWebhook credential in n8n UI**
1. Open n8n at http://localhost:5678
2. Settings → Credentials → Add Credential
3. Select **Custom Credentials** → **Client Credentials** (or "Generic Credential")
4. Fill in:
   - **Name:** `discordWebhook`
   - **URL:** Paste your Discord webhook URL (paste the FULL URL: `https://discord.com/api/webhooks/...`)
5. Click **Save**

**Step 3: Update credentials.json**
Add the `discordWebhook` credential to `n8n/workflows/credentials.json`:

```json
{
  "credentials": [
    {
      "name": "tftBackendApi",
      "type": "httpHeaderAuth",
      "description": "Bearer token auth for backend API. Created in n8n UI Settings -> Credentials."
    },
    {
      "name": "discordWebhook",
      "type": "genericCredentialType",
      "description": "Discord webhook URL for patch monitor failure notifications. Created in n8n UI Settings -> Credentials."
    }
  ]
}
```

<threat_model>

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| n8n → Riot CDN | External API call; Riot can return 403 or rate-limit |
| n8n → backend API | Internal Docker network call |
| n8n → Discord | Outbound webhook only; no PII transmitted |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-06-30 | Denial of Service | Riot CDN (403/timeout) | mitigate | On Riot CDN failure: Discord notification sent (not silent). n8n continues running; no infinite retry loop. |
| T-06-31 | Information Disclosure | Discord webhook URL | mitigate | Stored in encrypted n8n credential; webhook token is channel-scoped, low blast radius |
| T-06-32 | Tampering | patch_monitor.json | accept | Workflow JSON in local repo; user reviews before activating |

</threat_model>
