```json
    }
  },

  "settings": {},
  "id": "obsidian-ingest"
}
```

Key changes from the existing broken workflow:
- **Trigger:** Changed from `n8n-nodes-base.cron` (schedule) to `n8n-nodes-base.webhook` (manual)
- **URL:** Fixed from `http://backend:8000/ingest` → `http://backend:8000/api/ingest/obsidian`
- **Auth:** Added `authentication: "genericCredentialType"` + `genericAuthType: "httpHeaderAuth"` + `credentials.httpHeaderAuth: "tftBackendApi"`
- **Connection:** Updated from `Scheduled Trigger` → `Manual Webhook`

Note: The `Authorization: Bearer = Cre` in `headerParameters` is a placeholder that n8n replaces at runtime using the `tftBackendApi` credential. The credential `tftBackendApi` must be created in the n8n UI as type **"Header Auth"** with:
- Header Name: `Authorization`
- Header Prefix: `Bearer ` (with trailing space)
- Header Value: your `API_SECRET_KEY` value from `.env`

This task requires manual action in the n8n web UI to create the encrypted credential:

**Step 1: Open n8n UI**
Navigate to http://localhost:5678 and log in with `admin` / `tft-copilot`

**Step 2: Create the tftBackendApi credential**
1. Click the gear icon (Settings) in the top-right
2. Go to **Credentials**
3. Click **Add Credential**
4. Select **Header Auth** (or "Custom Credentials" → "Header Auth")
5. Fill in:
   - **Name:** `tftBackendApi`
   - **Header Name:** `Authorization`
   - **Header Prefix:** `Bearer ` (including the space)
   - **Header Value:** Your `API_SECRET_KEY` value from `.env`
6. Click **Save**

**Step 3: Create a placeholder credentials.json**
Write a minimal file `n8n/workflows/credentials.json` documenting the credential name:

```json
{
  "credentials": [
    {
      "name": "tftBackendApi",
      "type": "httpHeaderAuth",
      "description": "Bearer token auth for backend API. Created in n8n UI Settings -> Credentials."
    }
  ]
}
```

This file is documentation only — actual credentials are stored encrypted by n8n internally.
