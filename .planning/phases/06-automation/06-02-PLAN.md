<task type="auto">
  <name>Add n8n timezone env vars to docker-compose</name>
  <files>infra/docker-compose.yml</files>
  <read_first>
    infra/docker-compose.yml
  </read_first>
  <acceptance_criteria>
    - `GENERIC_TIMEZONE=Asia/Ho_Chi_Minh` appears under the n8n service `environment:` block
    - `N8N_PROXY_HOPS=1` appears under the n8n service `environment:` block
    - Backend service has `API_SECRET_KEY` env var (read from .env)
    - Existing n8n env vars (N8N_BASIC_AUTH_*, N8N_HOST, N8N_PROTOCOL) are preserved
    - Existing backend env vars (DATABASE_URL, OLLAMA_BASE_URL) are preserved
  </acceptance_criteria>
  <action>

Modify `infra/docker-compose.yml`:

1. Add to the `backend` service `environment:` block:
```yaml
    environment:
      - DATABASE_URL=postgresql://postgres:postgres@host.docker.internal:54322/postgres
      - OLLAMA_BASE_URL=http://host.docker.internal:11434
      - API_SECRET_KEY=${API_SECRET_KEY:-}
```

2. Add to the `n8n` service `environment:` block (preserve existing vars):
```yaml
    environment:
      - N8N_BASIC_AUTH_ACTIVE=true
      - N8N_BASIC_AUTH_USER=admin
      - N8N_BASIC_AUTH_PASSWORD=tft-copilot
      - N8N_HOST=localhost
      - N8N_PROTOCOL=http
      - GENERIC_TIMEZONE=Asia/Ho_Chi_Minh
      - N8N_PROXY_HOPS=1
      - API_SECRET_KEY=${API_SECRET_KEY:-}
      - DISCORD_WEBHOOK_URL=${DISCORD_WEBHOOK_URL:-}
```

Note: Use `${VAR:-}` syntax so Docker Compose reads from the .env file at runtime. The `:-` provides empty default if not set.

  </action>
  <verify>
    grep -n "GENERIC_TIMEZONE" infra/docker-compose.yml
    grep -n "N8N_PROXY_HOPS" infra/docker-compose.yml
    grep -n "API_SECRET_KEY" infra/docker-compose.yml
    grep -n "DISCORD_WEBHOOK_URL" infra/docker-compose.yml
  </verify>
  <done>n8n and backend services have all required env vars; timezone configured for Ho Chi Minh City</done>
</task>

<task type="auto">
  <name>Update env.example with automation secrets</name>
  <files>infra/.env.example</files>
  <read_first>

  </read_first>
  <acceptance_criteria>
    - `API_SECRET_KEY=<your-secret-here>` is present
    - `DISCORD_WEBHOOK_URL=<your-webhook-url>` is present
    - Existing n8n vars (WEBHOOK_URL, N8N_PROXY_HOPS, GENERIC_TIMEZONE, TZ) are preserved
    - Comment explains the purpose of each new var
  </acceptance_criteria>
  <action>

Modify `infra/.env.example` — add after the existing n8n Configuration section:

```bash
# API Auth (shared secret between n8n and backend)
# Generate a strong random string: python -c "import secrets; print(secrets.token_urlsafe(32))"
API_SECRET_KEY=<your-secret-here>

# Discord Webhook (for automation failure notifications)
# Create webhook in Discord: Channel Settings -> Integrations -> Webhooks
# Format: https://discord.com/api/webhooks/{webhook_id}/{webhook_token}
DISCORD_WEBHOOK_URL=<your-webhook-url>
```

Keep all existing entries intact (APP_ENV, OLLAMA_BASE_URL, DATABASE_URL, OBSIDIAN_VAULT_PATH, etc.).

  </action>
  <verify>

  </verify>
  <done>Environment template documents all required secrets with instructions for obtaining them</done>

<threat_model>

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| User → .env file | Secrets entered manually by user — treat as sensitive |
| Docker Compose → .env | Environment variables injected at container start |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-06-10 | Information Disclosure | API_SECRET_KEY | mitigate | Stored in .env (gitignored per project convention); never hardcoded in source files |
| T-06-11 | Information Disclosure | DISCORD_WEBHOOK_URL | mitigate | Stored in .env; Discord webhook tokens are scoped to a single channel |
| T-06-12 | Tampering | n8n env vars | accept | Local development environment; .env not committed to repo |

</threat_model>
