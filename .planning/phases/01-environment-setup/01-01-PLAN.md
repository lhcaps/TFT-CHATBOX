## Installation Steps

1. Download OllamaSetup.exe from https://ollama.com/download
2. Run the installer - Ollama installs as a Windows service
3. Verify installation:
   ```
   ollama -v
   ```
4. Start Ollama service if not running:
   ```
   ollama serve
   ```

## Model Pull Commands

Pull both models specified per D-02:
```
ollama pull qwen3:8b
ollama pull qwen3-embedding:4b
```

## GPU Verification

Run the following command and confirm VRAM usage:
```
ollama ps
```

Expected output:
- qwen3:8b with Processor: NVIDIA and size_vram > 0
- qwen3-embedding:4b with Processor: NVIDIA and size_vram > 0

If GPU not detected:
- Check NVIDIA drivers: `nvidia-smi`
- Verify CUDA 12.x installed
- Run Ollama from admin PowerShell

## Keep-Alive Configuration

Set default keep_alive to 15 minutes per D-04:
```
# Add to system environment or use in API calls
OLLAMA_KEEP_ALIVE=15m
```

## Supabase CLI Installation

1. Install via npm (requires Node.js):
   ```
   npm install -g supabase
   ```

2. Initialize Supabase in project root:
   ```
   cd d:/Study/Project/Tft Chatbox
   npx supabase init
   ```

3. Configure ports per D-06 (54322 for Postgres, 54323 for Studio):
   - Check supabase/config.toml for port configuration
   - Default ports should already match requirements

## Start Supabase

1. Ensure Docker Desktop is running (Supabase uses Docker internally)
2. Start Supabase local:
   ```
   npx supabase start
   ```

This launches:
- Postgres at port 54322
- Studio dashboard at port 54323
- API at port 54321 (optional)

## Verification

Run status command:
```
npx supabase status
```

Expected output:
