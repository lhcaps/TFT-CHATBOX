## Create .gitattributes

Create `d:/Study/Project/Tft Chatbox/.gitattributes`:

```
# Auto detect text files and perform LF normalization
* text=auto

# Python files - always use LF (per D-17)
*.py text eol=lf

# Shell scripts - always use LF
*.sh text eol=lf
*.bash text eol=lf

# Dockerfiles - always use LF
Dockerfile text eol=lf

# YAML and JSON - always use LF
*.yml text eol=lf
*.yaml text eol=lf
*.json text eol=lf

# Markdown - always use LF
*.md text eol=lf
*.mdx text eol=lf

# TypeScript/JavaScript - always use LF
*.ts text eol=lf
*.tsx text eol=lf
*.js text eol=lf
*.jsx text eol=lf

# CSS - always use LF
*.css text eol=lf

# Exclude binary files
*.png binary
*.jpg binary
*.jpeg binary
*.gif binary
*.ico binary
*.pdf binary
*.woff binary
*.woff2 binary
*.ttf binary
*.eot binary
```

## Why this matters (per D-17)

Windows uses CRLF (\r\n) by default, but:
- Python scripts fail with "bad interpreter" if shell sees CRLF
- Git shows entire file as changed when line endings differ
- Docker entrypoints fail if shell scripts have CRLF

This .gitattributes ensures:
- Python, shell, and config files always commit with LF
- Binary files are never converted
- Cross-platform consistency

## Enable Windows Long Path Support

Run the following PowerShell command as Administrator:

```powershell
# Enable LongPathsEnabled registry key (per D-18)
New-ItemProperty -Path "HKLM:\SYSTEM\CurrentControlSet\Control\FileSystem" `
    -Name "LongPathsEnabled" -Value 1 -PropertyType DWORD -Force

Write-Output "Windows Long Path support enabled"
```

## Verification

Check if enabled:
```powershell
(Get-ItemProperty -Path "HKLM:\SYSTEM\CurrentControlSet\Control\FileSystem").LongPathsEnabled
```

Expected output: 1

## What this fixes

- Prevents "Path too long" errors on Windows
- Allows npm/yarn to install in deeply nested node_modules
- Git operations work on nested paths
- Required for some build tools

## Alternative: Group Policy (if registry fails)

If registry modification is blocked:
1. Press Win+R, type `gpedit.msc`
2. Navigate to Computer Configuration > Administrative Templates > System > Filesystem
3. Enable "Enable Win32 long paths"

```python
"""Database connection pool management."""
from __future__ import annotations

import asyncpg
from typing import Optional

_pool: Optional[asyncpg.Pool] = None


async def get_pool() -> asyncpg.Pool:
    """Get or create the database connection pool."""
    global _pool
    if _pool is None:
        from app.config import settings
        _pool = await asyncpg.create_pool(
            settings.database_url,
            min_size=2,
            max_size=10,
        )
    return _pool


async def close_pool() -> None:
    """Close the database connection pool."""
    global _pool
    if _pool is not None:
        await _pool.close()
        _pool = None
```
