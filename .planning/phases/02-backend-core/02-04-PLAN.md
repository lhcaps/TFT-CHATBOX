```bash
SESSION_ID=$(curl -s -X POST http://localhost:8000/sessions \

```

**Step 2: Test streaming chat**
```bash
curl -N -X POST http://localhost:8000/chat \
  -H "Content-Type: application/json" \
  -d "{\"session_id\": \"$SESSION_ID\", \"message\": \"Hello, what is TFT?\", \"stream\": true}" \
  2>/dev/null | head -50
```

Expected output should contain SSE events in this format:
```
event: token
data: {"token": "T"}
event: token
data: {"token": "FT"}
...
event: done
data: {"done": true, "done_reason": "stop", "usage": {"prompt_tokens": ..., "completion_tokens": ..., "total_tokens": ...}}
```

**Step 3: Verify streaming format**
- Each token event should be: `event: token\ndata: {"token": "..."}`
- NOT raw text like: `data: Hello...`
- Done event should contain `done: true`, `done_reason`, and `usage` with token counts

**Step 4: Test non-streaming mode**
```bash
curl -X POST http://localhost:8000/chat \
  -H "Content-Type: application/json" \
  -d "{\"session_id\": \"$SESSION_ID\", \"message\": \"Hello\", \"stream\": false}"
```

Expected: JSON response with `{"text": "...", "usage": {...}, "done_reason": "stop", "citations": []}`
  </action>
  <done>Streaming and non-streaming chat work with structured events</done>
  <acceptance_criteria>
- [ ] SSE events have `event: token` and `data: {"token": "..."}` format (per D-04)
