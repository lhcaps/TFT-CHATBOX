  <files>apps/frontend/src/api/chat.ts</files>
  <action>
Create `apps/frontend/src/api/chat.ts`. This file contains two exports:

**1. `streamChat(opts, handlers)`** — SSE streaming client

- Uses `fetch` (not `EventSource`) so POST requests can be sent
- Accepts `ChatOptions` and an object with `onToken`, `onCitation`, `onDone`, `onError` callbacks
- Reads the response body with `ReadableStream` + `TextDecoder`
- Accumulates bytes in a buffer string
- Splits on `\n\n` to get SSE event boundaries
- For each complete event, parse `data: ` lines, join JSON, dispatch by type:
  - `token` → `onToken(token string)`
  - `citation` → `onCitation(citation object)`
  - `done` → `onDone(usage object)`
  - `error` → `onError(Error)`
- On stream end, flush any remaining buffer (in case the last event didn't end with `\n\n`)
- On `AbortError`, swallow the error (don't call `onError`)
- Uses `AbortController.signal` passed via `ChatOptions.signal`

**2. `chatNonStreaming(opts)`** — non-streaming fallback

- Same POST but `stream: false`
- Returns parsed JSON response

Internal helpers:
- `extractEvents(buffer: string)` → splits on `\n\n`, returns `{ events: Event[], remainder: string }`
- `parseSSEPart(raw: string)` → parses a single SSE block, returns `Event | null`
- `processChunk(chunk: string, handlers)` → flushes a buffer on stream end
- `Event` union type: `{type:'token',token:string}` | `{type:'citation',citation:Record}` | `{type:'done',usage:Record}` | `{type:'error',message:string}`

Note: `data:` lines can span multiple lines if JSON is broken across lines — join all `data:` values before parsing.

const BASE = '/api';

export async function listSessions(): Promise<Session[]> {
  const res = await fetch(`${BASE}/sessions`);

  return res.json();
}

export async function createSession(title?: string, mode: string = 'normal'): Promise<Session> {
  const res = await fetch(`${BASE}/sessions`, {
    method: 'POST',
    headers: { 'Content-Type': 'application/json' },
    body: JSON.stringify({ title, mode }),
  });
  if (!res.ok) throw new Error(`Failed to create session: ${res.status}`);
  return res.json();
}

export async function getSession(id: string): Promise<Session> {
  const res = await fetch(`${BASE}/sessions/${id}`);
  if (!res.ok) throw new Error(`Failed to get session: ${res.status}`);
  return res.json();
}

export async function deleteSession(id: string): Promise<void> {
  const res = await fetch(`${BASE}/sessions/${id}`, { method: 'DELETE' });
  if (!res.ok) throw new Error(`Failed to delete session: ${res.status}`);
}

export async function getSessionMessages(sessionId: string): Promise<MessageOut[]> {
  const res = await fetch(`${BASE}/sessions/${sessionId}/messages`);
  if (!res.ok) throw new Error(`Failed to get session messages: ${res.status}`);
  return res.json();
}

Create `apps/frontend/src/hooks/useStreamingMessages.ts`. This hook owns the chat message state, streaming state, and error state. It accepts session history as a prop.

**Props:** `history: MessageOut[]` — persisted messages from the API (passed from `useSession`)

**State:**
- `messages: Message[]` — all messages including user and assistant (local format with `isStreaming`)
- `isStreaming: boolean`
- `error: string | null`

**Refs:**
- `acRef` — the current AbortController
- `tokenAccRef` — accumulates the full assistant response text
- `endedRef` — guard to prevent double-calling callbacks

**`toMessages(apiMessages: MessageOut[]): Message[]`:**
Convert API message format to local `Message[]` format (drops `isStreaming`, adds it only for live streaming messages):

```typescript
function toMessages(apiMessages: MessageOut[]): Message[] {
  return apiMessages.map((m) => ({
    role: m.role,
    content: m.content,
    citations: m.citations,
    isStreaming: false,
  }));
}
```

**`useEffect([history, isStreaming])`:**
When `history` changes and not streaming: update `messages` from `toMessages(history)`.

**Core logic (`sendMessage`):**
1. Guard: if already streaming, ignore
2. Reset `endedRef.current = false` and `tokenAccRef.current = ''`
3. Clear error
4. Set `isStreaming = true`
5. Create user message and assistant placeholder (`isStreaming: true`)
6. Add both to messages
7. Create new `AbortController`
8. Call `streamChat` with callbacks

**onToken:**
- Append to `tokenAccRef.current`
- Update last message content

**onCitation:**
- Append citation to last message's citations array

**onDone:**
- Guard with `endedRef` — return if already called
- Set `endedRef.current = true`
- Update last message: `isStreaming: false`, content from `tokenAccRef`
- Set `isStreaming = false`

**onError:**
- Guard with `endedRef` — return if already called
- Set `endedRef.current = true`
- If `AbortError`: keep accumulated text, mark last message as not streaming
- Otherwise: set `error`, update last message to show error text
- Set `isStreaming = false`

**abort():** `acRef.current?.abort()`
**clearMessages():** set messages to `[]`
**clearError():** set error to `null`

**Cleanup:** `useEffect` returns `() => acRef.current?.abort()` to stop stream on unmount.
