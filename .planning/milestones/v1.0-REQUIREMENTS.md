## v1 Requirements — All Complete

### Foundation

- [x] **FOUND-01**: Ollama runs natively on Windows, serving chat model (qwen3:8b → qwen3:1.7b) and embedding model (qwen3-embedding:4b) at localhost:11434
- [x] **FOUND-02**: Supabase local CLI launches Postgres + pgvector/HNSW at port 54322 with dashboard at 54323
- [x] **FOUND-03**: Docker Compose orchestrates backend + frontend + n8n; Ollama and Supabase run as host services
- [x] **FOUND-04**: Health endpoint checks both Ollama and Supabase connectivity

### Backend

- [x] **BACK-01**: FastAPI app exposes GET /health, POST /sessions, GET /sessions, POST /chat, POST /search, POST /ingest/obsidian, POST /ingest/tft-static
- [x] **BACK-02**: POST /chat with stream=true returns SSE with token events and done event with usage stats; stream=false returns complete JSON
- [x] **BACK-03**: Session persistence: sessions + messages tables with proper indexes; messages persisted to DB and retrievable by session_id
- [x] **BACK-04**: CORS configured to allow http://localhost:5173 explicitly (no wildcard)

### Frontend

- [x] **FRONT-01**: React + Vite + Tailwind chat UI scaffolded and running at localhost:5173
- [x] **FRONT-02**: Chat shell renders messages with user/assistant distinction, mode tabs, and input area
- [x] **FRONT-03**: SSE streaming: tokens appear in real-time using fetch + ReadableStream pattern
- [x] **FRONT-04**: Abort controller stops generation mid-stream
- [x] **FRONT-05**: Mode toggle switches between Normal / RAG / Coach modes

### Database Schema

- [x] **DB-01**: Tables: chat_sessions (uuid, title, mode, metadata, timestamps), chat_messages (bigserial, session_id, role, content, citations, usage, timestamps)
- [x] **DB-02**: Tables: documents (uuid, source_type, source_path, source_hash, title, season, patch, locale, metadata, raw_markdown, timestamps), document_chunks (bigserial, document_id, chunk_index, heading_path, content, token_estimate, embedding vector(1024), metadata, fts tsvector, timestamps)
- [x] **DB-03**: HNSW index on document_chunks.embedding with vector_cosine_ops
- [x] **DB-04**: GIN index on document_chunks.fts
- [x] **DB-05**: hybrid_search_chunks() SQL function with RRF (semantic_weight=2, full_text_weight=1)
- [x] **DB-06**: B-tree indexes on session_id + created_at, document_id + chunk_index

### RAG Pipeline

- [x] **RAG-01**: Obsidian vault ingest: walk .md files (skip .obsidian/), parse frontmatter, split by heading hierarchy
- [x] **RAG-02**: Chunking: 2000 char chunks with 500 char overlap, preserving heading path as metadata
- [x] **RAG-03**: Batch embedding via Ollama /api/embed with dimensions=1024, batch size 16
- [x] **RAG-04**: Hash-based incremental re-ingest: source_hash on documents, DELETE before re-INSERT on changed files
- [x] **RAG-05**: TFT static data ingestion: champions, traits, items, augments from Riot Data Dragon + CommunityDragon + official patch page scraping
- [x] **RAG-06**: Metadata filtering: patch and season columns enable scoped retrieval
- [x] **RAG-07**: Basic citation display: sources stored in citations JSONB field, rendered at end of response

### Prompt Modes

- [x] **PROMPT-01**: Normal mode prompt: concise, factual, admits uncertainty
- [x] **PROMPT-02**: RAG mode prompt: grounds answers in retrieved context, cites [doc:title > heading]
- [x] **PROMPT-03**: Coach mode prompt: 2-3 lines of play with econ/tempo/cap/pivot framing, no dictating, no real-time scouting references

### Automation

- [x] **AUTO-01**: n8n workflow: Schedule Trigger every 4 hours → POST /api/ingest/obsidian
- [x] **AUTO-02**: n8n workflow: Schedule Trigger every 6 hours → check Riot versions.json → POST /api/ingest/tft-static if patch changed
- [x] **AUTO-03**: ngrok tunnel for webhook exposure; script auto-refreshes URL on tunnel restart
- [x] **AUTO-04**: n8n configured with GENERIC_TIMEZONE=Asia/Ho_Chi_Minh and N8N_PROXY_HOPS=1

### DevOps / Polish

- [x] **POLY-01**: Query embedding cache: LRU with 30-minute TTL keyed by (query, mode, patch)
- [x] **POLY-02**: Static TFT JSON on-disk cache with patch version check
- [x] **POLY-03**: Smoke test: 20-question eval set covering comp, item, augment, pivot scenarios
- [x] **POLY-04**: GPU memory monitoring: frontend displays current VRAM usage via Ollama /api/ps
- [x] **POLY-05**: Windows path handling: pathlib used throughout, long paths enabled, CRLF/LF handled
