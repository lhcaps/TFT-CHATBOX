## Overview

Full-stack local-first AI copilot for Teamfight Tactics (TFT Set 17 Space Gods). Three chat modes — Normal, RAG, Coach — powered by Ollama local LLM, Supabase pgvector RAG, and Obsidian knowledge source. Runs entirely on the user's machine (Windows), no cloud APIs.

## Key Decisions

- Ollama native (not containerized): GPU passthrough on Windows
- 1024-dim embeddings: qwen3-embedding:4b, HNSW index limit respected
- Supabase local CLI: Postgres + pgvector without Docker complexity
- 3-mode chat (Normal/RAG/Coach): TFT policy compliant
- SSE for streaming: sufficient for one-way token streaming
- In-memory LRU cache: MVP scale
- Bearer token auth: Defense in depth
- ngrok dropped: 100% local operation

## Issues Resolved

- Smoke test endpoint URLs missing /api prefix → fixed to /api/chat and /api/health/gpu
- VARCHAR(16) session_id too short for UUIDs → migrated to VARCHAR(64)
- ForeignKeyViolationError on chat (session not pre-created) → auto-create session in chat endpoint
- Cache key unbounded length → truncated to MAX_QUERY_LENGTH = 512
- Embedding dims mismatch (qwen3-embedding:4b creates 2560 dims, DB limit 1024) → truncated to 1024 dims in retrieval
- RAG 500 errors (metadata stored as JSON string, not dict) → JSON parse in retrieval service
- Riot CDN 403 errors → added User-Agent header, fallback to CommunityDragon + official patch page scraping

## Issues Deferred

- n8n workflows not yet activated (user action required)
- Obsidian vault path needs user-specific configuration
- RLS Row Level Security disabled on DB tables (security advisory)

## Technical Debt Incurred

- qwen3-embedding:4b creates 2560-dim embeddings (must truncate to 1024 for DB) — consider a 1024-dim embedding model
- qwen3:8b replaced with qwen3:1.7b for 12GB VRAM compatibility — quality tradeoff on response length
