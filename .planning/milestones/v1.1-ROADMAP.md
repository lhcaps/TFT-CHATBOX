## Key Decisions

- Patch state in DB (not file) — queryable, testable, single source of truth
- Dedicated `POST /api/ingest/patch-notes` endpoint — clean separation from static data ingest
- Patch URL discovery via listing page scrape — falls back to URL pattern template
- Frontend "Check for updates" button (manual) vs auto-refresh (too aggressive for local app)
- Discord: verbose notifications (start + success/fail), not just new-patch detection
